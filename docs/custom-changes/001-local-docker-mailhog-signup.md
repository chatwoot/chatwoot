# 001 - Local Docker, Mailhog, And Signup Defaults

## Branch

```text
development/local-docker-mailhog-signup
```

## Purpose

This change prepares the forked Chatwoot project for local Brain Assistant development with Docker, Mailhog email capture, and account signup enabled by default.

## Changes Made

### Docker Compose

File: `docker-compose.yaml`

- Updated Postgres to read `POSTGRES_PASSWORD` from `.env`.
- Added `docker/entrypoints/rails.sh` as the Sidekiq entrypoint.
- This lets Sidekiq wait for Postgres and install missing gems before starting.

### Rails Entrypoint

File: `docker/entrypoints/rails.sh`

- Added startup sync for `ENABLE_ACCOUNT_SIGNUP`.
- If `.env` contains `ENABLE_ACCOUNT_SIGNUP=true`, the entrypoint writes that value into Chatwoot's `InstallationConfig` table.
- This keeps signup enabled after restarts, even after Chatwoot creates its database-backed config row.

### Vite Entrypoint

File: `docker/entrypoints/vite.sh`

- Added `bundle check || bundle install` before starting Vite.
- This prevents missing Ruby gem errors when Vite runs in a fresh Docker volume.

### Installation Config

File: `config/installation_config.yml`

- Changed the default value of `ENABLE_ACCOUNT_SIGNUP` from `false` to `true`.

### Generated Annotation And Schema Churn

Files:

- `app/models/platform_banner.rb`
- `db/schema.rb`

These were generated by Chatwoot setup commands. They are not business logic changes.

## Local Environment

The local `.env` is not committed because Chatwoot ignores it. For local setup, create `.env` with these important values:

```env
FRONTEND_URL=http://localhost:3000
ENABLE_ACCOUNT_SIGNUP=true
POSTGRES_PASSWORD=chatwoot_pg_local
REDIS_URL=redis://:chatwoot_redis_local@redis:6379
REDIS_PASSWORD=chatwoot_redis_local
SMTP_ADDRESS=mailhog
SMTP_PORT=1025
MAILER_INBOUND_EMAIL_DOMAIN=chatwoot.local
RAILS_INBOUND_EMAIL_SERVICE=relay
```

## Local Ports

| Port | Service |
| --- | --- |
| `3000` | Chatwoot Rails |
| `3036` | Chatwoot Vite |
| `5432` | Postgres |
| `6379` | Redis |
| `1025` | Mailhog SMTP |
| `8025` | Mailhog UI |

## Verification

After applying the changes:

```sh
docker compose up -d postgres redis mailhog
docker compose run --rm rails bundle exec rails db:chatwoot_prepare
docker compose up -d rails sidekiq vite
```

Expected checks:

- `http://localhost:3000/` returns `200`.
- `http://localhost:3000/app/login` returns `200`.
- `http://localhost:3000/app/auth/signup` renders with `signupEnabled: 'true'`.
- `http://localhost:8025` opens Mailhog.
