## Current State Summary

Both features are **functionally complete end-to-end** — template CRUD + Meta submission + sync + sending in conversations, and flows CRUD + Meta API + visual builder + flow response handling. The gaps are primarily in **UI polish, UX friction, and missing WhatsApp API capabilities**.

---

## Message Templates — Improvement Plan

### A. List Page (`MessageTemplatesListPage.vue`)

**1. Search & Filters** (High Impact)
- Add a text search input to filter templates by name
- Add status filter chips/dropdown (Approved, Pending, Rejected, Draft, Paused, Disabled)
- Add category filter (Marketing, Utility, Authentication)
- Add language filter

**2. Pagination** (High Impact)
- The list renders ALL templates in a single `v-for` — will degrade with 100+ templates
- Add pagination component or virtual scrolling

**3. Sortable Columns** (Medium)
- Make table headers clickable to sort by name, category, language, status, updated_at
- Add sort direction indicators (chevron up/down)

**4. Empty/Filtered State** (Medium)
- Currently no "no results" state when inbox filter yields zero matches — user sees a blank table body
- Add a "No templates match your filters" empty state with a "Clear filters" action

**5. Quick Preview** (Medium)
- Add an eye icon action to preview a template in a modal without entering the editor
- Reuse `TemplatePreview.vue` in a dialog

**6. Rejection Reason Display** (Medium)
- Meta returns rejection reasons — surface them in a tooltip or expandable row for rejected templates
- Requires storing the rejection reason from the API response (currently not persisted)

**7. Template Clone/Duplicate** (Low-Medium)
- Add a "Duplicate" action that pre-fills the builder with an existing template's data (new name suffix `_copy`)
- Useful for creating language variants or similar templates

**8. Bulk Actions** (Low)
- Add row checkboxes, select-all, and bulk delete
- Less critical since templates are usually managed individually

**9. Sync UX** (Low)
- Add a tooltip explaining what "Sync" does
- Show last sync timestamp per inbox
- Show a sync progress indicator

### B. Builder Page (`MessageTemplateBuilderPage.vue`)

**1. Per-Field Validation Errors** (High Impact)
- Currently the submit button just stays disabled with no indication of *why*
- Show inline error messages under each field: name (format, uniqueness), inbox (required), body (required, length), etc.
- Show a validation summary at the top when submission is attempted

**2. Unsaved Changes Warning** (High Impact)
- No dirty-state detection — "Discard" and browser back navigate away silently
- Add `beforeRouteLeave` guard with a confirmation dialog when form is dirty

**3. Rich Text Formatting Toolbar** (High Impact, UX differentiator)
- WhatsApp templates support `*bold*`, `_italic_`, `~strikethrough~`, `` `monospace` ``
- Add formatting buttons above the body textarea that insert the markers
- The preview should also render these formatting marks

**4. Language List Expansion** (Medium)
- Currently hardcoded to 20 languages; WhatsApp supports ~65
- Fetch supported languages from configuration or expand the list to cover all Meta-supported locales

**5. Save as Draft vs. Submit** (Medium)
- Currently a single "Create" action that immediately submits to Meta
- Allow saving locally as draft without Meta submission, then a separate "Submit for Review" action
- Useful for preparing templates before they're ready

**6. Loading State for Edit Mode** (Medium)
- When editing, the form loads empty then fills — no loading skeleton or spinner
- Show a skeleton/spinner while fetching template data

**7. Character Counters** (Medium)
- Add real-time character counters for: template name (max 512), header text (max 60), body (max 1024), footer (max 60), button labels (max 25)
- Show as `X/max` counters on the right side of each input

**8. Responsive Layout Breakpoint** (Low-Medium)
- The side-by-side preview only works at `xl` (1280px+) — most laptops lose it
- Lower the breakpoint to `lg` (1024px) or make the preview a floating/dockable panel

**9. Button Ordering Transparency** (Low)
- Buttons auto-sort by category (CTA first, QR last) without telling the user
- Add a visual separator between CTA and Quick Reply groups with labels
- Add drag-and-drop reordering within each group

## WhatsApp Flows — Improvement Plan

### E. Flows List Page (`WhatsappFlowsListPage.vue`)

**1. Design System Alignment** (High Impact)
- Uses raw `<button>` elements and a custom modal instead of `Button` and `Dialog` components
- Align with the templates list page's design patterns for visual consistency

**2. Publish Confirmation** (High Impact)
- Publishing is **irreversible** (flows cannot be unpublished) — yet there's no confirmation dialog
- Add a confirmation dialog: "Publishing is permanent. This flow cannot be edited after publishing."

**3. Search, Filters, Pagination** (Medium)
- Same gaps as templates list — no search, no status filter, no pagination
- Add text search, status filter chips, and pagination

**4. Flow Preview from List** (Medium)
- Add a "Preview" action that opens the Meta preview URL in a modal/iframe
- The backend `preview` endpoint already exists

**5. Flow Clone** (Medium)
- Meta supports cloning flows via `clone_flow_id`
- Add a "Duplicate" action that creates a copy of the flow

**6. Metadata Display** (Low)
- Show created_at, updated_at, categories, and flow_id in the list
- Show validation errors if the flow has any

### F. Flow Builder (`WhatsappFlowBuilderPage.vue`)

**1. Screen Navigation Graph** (High Impact, UX differentiator)
- This is mentioned in the existing improvements doc (section 4, item 6) as a desired feature
- Add a visual graph/diagram showing screen-to-screen navigation (routing model)
- Could be a minimap or a separate "Navigation" tab
- Show directed arrows between screens based on Footer navigation actions

**2. Unsaved Changes Detection** (High Impact)
- Same issue as templates — no dirty-state detection, no confirmation on navigate-away
- Add `beforeRouteLeave` guard

**3. Screen Editing (Title & ID)** (High — currently broken)
- `FlowScreenList.vue` only emits `select`, `add`, `remove` — never emits `update-title` or `update-id`
- The handlers in the builder page (`updateScreenTitle`, `updateScreenId`) are dead code
- Implement inline editing of screen title and ID in the screen list

**4. Undo/Redo** (Medium-High)
- A visual builder without undo/redo is risky — accidental deletions are permanent
- Implement a simple undo stack (snapshot-based) with Ctrl+Z/Ctrl+Y keybindings

**5. Flow Validation Button** (Medium-High)
- Add a "Validate" button that checks the flow JSON against WhatsApp's requirements before saving
- Show validation errors inline on the canvas (red badges on problematic components/screens)

**6. Responsive Panel Layout** (Medium)
- Fixed 3-panel layout is unusable on smaller screens
- Add collapsible sidebars or a drawer pattern for the screen list and property panel
- When no component is selected, the property panel disappears causing layout shift — show a placeholder state instead

**7. Screen Reordering & Duplication** (Medium)
- Screens cannot be reordered or duplicated
- Add drag-and-drop reordering in the screen list
- Add a "Duplicate Screen" action

**8. Component Drag-and-Drop** (Medium)
- Currently uses move up/down buttons only — no drag-and-drop reordering
- Move buttons don't work on touch devices (hover-only visibility)
- Implement drag-and-drop with a grip handle

**9. Import/Export Flow JSON** (Low-Medium)
- Add "Download JSON" and "Import JSON" buttons for developers who prefer editing flows externally
- Useful for version control and sharing flows between environments

**10. Auto-Save / Draft Recovery** (Low)
- localStorage-based auto-save to recover work if the browser crashes

### G. Flow Canvas (`FlowCanvas.vue`)

**1. Component Delete Confirmation** (High)
- Single click deletes a component permanently with no confirmation and no undo
- Add a brief confirmation or undo toast

**2. Phone Frame Fidelity** (Medium)
- Missing device bezel, rounded corners, status bar (time/battery/signal)
- Missing WhatsApp header avatar and online status
- Adding these makes the preview feel more realistic and trustworthy

**4. Truncated Options Indicator** (Low)

---

### H. Flow Property Panel (`FlowPropertyPanel.vue`)

**1. Input Debouncing** (High)
- Every keystroke triggers `emitUpdate()` — no debounce
- Add 300ms debounce on text inputs to reduce unnecessary re-renders

**2. Footer Navigation Bug** (High — logic error)
- `otherScreens` filter has `|| true` that makes it include ALL screens including the current one
- Users can create infinite navigation loops by selecting the current screen as the target
- Fix: `props.screens.filter(s => s.id !== currentScreenId)`

**3. Data Source Option ID Uniqueness** (Medium)
- `addOption` generates sequential IDs without uniqueness validation
- Removing and re-adding options creates duplicate IDs
- Use UUID or validate uniqueness

**4. Missing Properties** (Medium)
- `visible` — conditional visibility based on data variables
- `init-value` — initial/default values for inputs
- `min-date` / `max-date` for DatePicker
- `max-length` for TextInput and TextArea
- `enabled` for conditional enable/disable

**5. Design System Alignment** (Low-Medium)
- The required toggle is a custom-built switch instead of the design system `Switch` component
- Labels are not associated with inputs via `for`/`id`

## New Feature Opportunities

### J. Entirely New Capabilities

**1. WhatsApp Catalog/Product Messages** (High Impact)
- Already mentioned in the improvements doc as a future item
- Meta's Commerce API supports: Single Product Message (SPM), Multi-Product Message (MPM), Catalog button in templates
- Requires: Product catalog sync, product picker UI, product message composer in conversation view
- High value for e-commerce use cases

**2. Template Send Analytics Dashboard** (Medium-High)
- Aggregate view of template performance: sent/delivered/read/failed counts over time
- Per-template and per-inbox breakdowns
- Uses Meta's Analytics API (`GET /{WABA_ID}/template_analytics`)

**3. Bulk Template Sending UI** (Medium)
- The backend `Whatsapp::OneoffCampaignService` already exists (gated by `whatsapp_campaign` flag)
- Build a proper UI: recipient selection (by label/segment), template picker, variable mapping, scheduling, delivery report
- Currently there's no frontend for this

**4. WhatsApp Business Profile Editor** (Medium)
- Allow editing business profile (about, description, address, website, category) from inbox settings
- The health dashboard already shows read-only profile info
- Use `POST /{phone_number_id}/whatsapp_business_profile` API

**5. Flow Templates Gallery** (Low-Medium)
- Pre-built flow templates for common use cases: lead generation, appointment booking, surveys, support intake
- Users pick a template and customize it instead of building from scratch
- Could be local JSON templates or use Meta's clone API

**6. Interactive Message Composer** (Low-Medium)
- Allow agents to send interactive messages (button lists, reply buttons, CTA URLs) directly from the conversation composer — not just templates
- Currently only possible via templates or the AI agent
- Only works within the 24h service window
